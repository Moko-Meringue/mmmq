# 메시지 큐란?

생산자와 소비자가 "메시지"를 통해 소통할 수 있는 아키텍처(개념)를 말한다.
서버 간 통신이 필요할 때, 다른 서버에 직접 의존하지 않고 미들웨어(메시지 큐)에 의존한다.
메시지 큐를 통해 메시지와 소비자 간의 유연한 상호작용을 구현할 수 있다.

## 비동기(Asynchronous)

- 사용자에게 빠른 응답을 줄 수 있다.
    - 서버 내부적으로 오래 걸리는 작업을 수행해야 할 때, 사용자를 기다리게 하지 않고 비동기적으로 응답을 줄 수 있다.

## 낮은 결합도

- 생산자와 소비자를 분리해, 자신의 일에만 집중할 수 있게 된다.
- 생산자는 자신의 행동으로 인해 수행되어야 할 부가적인 작업들을 신경쓰지 않고, 그저 "내가 수행한 작업 내용"만 메시지 큐에 제출하면 된다.
- 소비자는 누가 생산자인지는 알 필요 없이 작업을 수행한다.

## 확장성

- 생산자와 소비자는 서로의 클러스터에 대해 직접적으로 의존하지 않기 때문에 각 클러스터는 확장에 유연하다.

## 신뢰성

- 소비자 서비스가 다운되더라도 작업이 유실되지 않고 보존되어 메시지큐에 삽입한 작업이 반드시 수행됨을 신뢰할 수 있다.
- 다만, 메시지 자체의 구체적인 성공 여부는 보장할 수 없다. (애초에 잘못된 요청일 수도 있으므로)
